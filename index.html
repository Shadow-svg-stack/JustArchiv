Guide de Configuration Supabase

1 Créer un projet Supabase

• Allez sur supabase.com

• Cliquez sur "Start your project" puis "New project"

• Choisissez un nom et un mot de passe pour votre base de données

• Sélectionnez une région proche de vous

• Cliquez sur "Create new project"

2 Récupérer les clés d'API

• Dans votre projet, allez dans "Settings" → "API"

• Copiez l'URL du projet (Project URL)

• Copiez la clé anonyme (anon/public key)

Exemple d'URL :

https://abcdefghijklmnop.supabase.co

Exemple de clé anonyme :

eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpc3MiOiJzdXBhYmFzZSI...

3 Configurer le code

• Ouvrez le fichier HTML dans un éditeur de code

• Trouvez les lignes 41-42 dans le code

• Remplacez les valeurs par vos vraies informations :

// Remplacez ces valeurs :
const supabaseUrl = 'https://VOTRE-PROJET-ID.supabase.co';
const supabaseKey = 'eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...';

4 Créer les tables de base de données

• Dans Supabase, allez dans "Table Editor"

• Créez les tables suivantes avec le SQL ci-dessous :

Table "documents" :
CREATE TABLE documents (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE,
  title TEXT NOT NULL,
  description TEXT,
  category TEXT,
  type TEXT,
  size BIGINT,
  file_url TEXT,
  file_path TEXT,
  tags TEXT[],
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);
Table "categories" :
CREATE TABLE categories (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE,
  name TEXT NOT NULL,
  color TEXT DEFAULT 'blue',
  icon TEXT DEFAULT 'folder',
  count INTEGER DEFAULT 0,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

5 Configurer le stockage (optionnel)

• Allez dans "Storage" dans Supabase

• Créez un bucket nommé "documents"

• Configurez les politiques de sécurité selon vos besoins

Sécurité importante

⚠️ Ne jamais exposer votre clé secrète (service_role) !

• Utilisez uniquement la clé anonyme (anon/public) dans le frontend

• Configurez les Row Level Security (RLS) dans Supabase

• En production, utilisez des variables d'environnement
